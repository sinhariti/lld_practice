# LLD Practice

Applying SOLID Principles and practicing Low-Level Design in C++

## 🎯 SOLID Principles

**S** - **Single Responsibility**: Each class should have one job  
**O** - **Open/Closed**: Open for extension, closed for modification  
**L** - **Liskov Substitution**: Subtypes must be substitutable for their base types  
**I** - **Interface Segregation**: Many specific interfaces are better than one general interface  
**D** - **Dependency Inversion**: Depend on abstractions, not concrete implementations

## 📂 Repository Structure

Each branch contains one LLD question with two implementations:

```
branch-name/
├── file-code.cpp    # Normal code implementation
└── file-solid.cpp   # Code with SOLID principles applied
```

**file-code.cpp**: Standard implementation without SOLID principles  
**file-solid.cpp**: Refactored implementation demonstrating SOLID principles
